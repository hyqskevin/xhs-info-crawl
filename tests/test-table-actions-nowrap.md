# 测试用例：表格操作列不换行

## 测试环境
- 前端：Vitest + Vue Test Utils
- 验证：Element Plus ElTableColumn min-width + class="action-column"

## 关联
- spec：`docs/superpowers/specs/2026-07-17-table-actions-column-nowrap-design.md`

## 验收案例

### TC-TAW-001：活动管理操作列
**前置**：活动列表页加载。

**操作**：浏览器宽度 1280px。

**预期**：
- "编辑"/"删除"按钮单行展示
- 不出现换行（按钮文字不被裁切到下一行）

### TC-TAW-002：博主管理操作列
**前置**：博主列表页加载。

**操作**：浏览器宽度 1280px。

**预期**：
- "编辑"/"删除"按钮单行展示
- 即使加了"补充博主信息"按钮（profile_url 为空时），也单行展示

### TC-TAW-003：任务管理操作列
**前置**：任务列表页加载。

**操作**：浏览器宽度 1280px。

**预期**："查看日志"按钮单行展示。

### TC-TAW-004：去重管理操作列
**前置**：去重列表页加载。

**操作**：浏览器宽度 1280px。

**预期**："合并A"/"合并B"/"忽略"按钮单行展示。

### TC-TAW-005：报告管理操作列
**前置**：报告列表页加载。

**操作**：浏览器宽度 1280px。

**预期**："下载MD"/"下载XLSX"按钮单行展示。

### TC-TAW-006：移动端允许折行
**操作**：浏览器宽度 600px。

**预期**：操作列可折行，但按钮仍可点击（不被遮挡）。

## 自动化覆盖

通过 CSS 类 `action-column` 配合 `white-space: nowrap` 实现，
具体单元测试覆盖在 `frontend/src/views/` 各组件 spec 中（按钮存在 + 列宽足够）。

## 手动验收

1. 浏览器宽度 1280px。
2. 逐个访问活动/博主/任务/去重/报告/配置中心页面。
3. 每个列表页"操作列"按钮单行展示，无文字换行。
4. 移动端 600px 宽度，操作列可折行但按钮可点。
