# 抓取执行权与安全停止验收

## 组件与 API 案例

1. 创建任务后返回并持久化 `run_token`，Celery 消息携带相同令牌。
2. 重启任务必须更换令牌。
3. 陈旧令牌、重复消息和非 `PENDING` 任务无法领取执行权。
4. PID 注册以 `task_id + run_token` 隔离，新旧执行互不覆盖。
5. 在详情返回时写入 `STOP_REQUESTED`，断言不再下载、不创建 Note，最终状态为 `STOPPED`。

## 浏览器案例

1. 在仪表盘启动耗时抓取。
2. 点击“停止抓取”，观察“安全停止中”和阶段进度。
3. 状态变为 `STOPPED` 后继续抓取。
4. 记录旧任务的日志、计数和数据数量；新任务运行期间旧任务不得再变化。

## 通过标准

- 自动化所有权与停止测试通过；
- 重复/陈旧消息不重跑历史任务；
- 停止后旧执行不再写数据库或归档文件。
