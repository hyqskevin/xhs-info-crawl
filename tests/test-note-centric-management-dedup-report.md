# 推文维度管理、去重与周报验收

## 后端案例

1. 同一推文使用 `/explore/{id}` 与 `/discovery/item/{id}`，并更换 `xsec_token`，搜索结果只保留一条。
2. 数据库已有相同 `platform_note_id` 时，在详情、下载、OCR 前跳过，且不增加失败计数。
3. 推文列表一行返回一个 Note，并返回有效子活动数量。
4. 推文详情返回正文、原文链接、图片 OCR 和全部有效子活动。
5. 推文级批量审核不篡改子活动识别状态；批量删除后整篇推文退出列表。
6. 模糊去重候选使用 Note A/B，候选对幂等；保留一方后另一方及其子活动退出业务列表。
7. 周报按推文发布时间与单城市筛选，只包含已审核推文，并携带推文下所有未删除活动。
8. Markdown 一篇推文一个章节；Excel 一篇推文一行，包含原文与图片链接。

## 前端与浏览器案例

1. 活动管理首页显示推文标题、城市、发布时间、识别活动数和审核状态。
2. 详情宽抽屉在信息表格后显示子活动和来源图片。
3. 子活动支持编辑、删除；推文支持批量审核、删除。
4. 去重审核展示推文 A/B，保留后刷新候选列表。
5. 周报城市为单选、周次使用日期组件，并可下载 Markdown 和 Excel。

## 通过标准

- 后端、前端组件测试和 Playwright E2E 通过；
- 同一历史推文更换 token 后不产生重复 Note、Activity、图片或失败日志；
- 一篇多活动推文在首页和 Excel 中均只占一行。
