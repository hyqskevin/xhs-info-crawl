# 配置测试、周报导出与活动批量删除设计

## 目标

在不改变一期 Vue 3、Element Plus、FastAPI、SQLite 技术栈的前提下，改善三处管理端交互：明确展示 OpenCLI 连接测试过程；以日期和城市配置选择周报范围并可靠下载文件；支持对勾选的活动执行批量软删除。

## 全局约束

- 前端继续使用 Element Plus 组件和 `@element-plus/icons-vue` 图标，不使用 Emoji，不自行实现已有基础 UI。
- 周报一次只允许选择一个城市，不支持多城市合并导出。
- 城市选项只来自配置中心中已启用的城市，不允许填写内部城市代码。
- 活动删除保持现有软删除语义，即状态更新为 `DELETED`，不物理删除数据库记录。
- 所有新增交互同时补充组件测试、后端 API 测试和 Playwright 浏览器功能测试，并同步更新 `tests` 目录规格。

## 1. OpenCLI 测试状态

配置中心保留“测试 OpenCLI”按钮。点击后设置本地 `testingOpenCLI` 状态：

- 按钮在请求期间禁用，防止重复测试。
- 按钮右侧显示 Element Plus `Loading` 图标，并使用 `is-loading` 动画；测试结束后立即隐藏。
- 请求成功后显示成功 Toast：“OpenCLI 登录与连接正常”。
- 未登录时显示错误 Toast：“请在 Chrome 登录小红书”；其他错误优先展示接口返回原因，没有原因时展示连接测试失败提示。
- 无论成功或失败，都在 `finally` 中恢复按钮和加载图标状态。

该功能不改变现有 `POST /api/v1/settings/opencli/test` 接口。

## 2. 单城市周报生成与下载

### 2.1 生成条件

周报工具栏改为：

- Element Plus 周选择器 `ElDatePicker type="week"`，用户选择自然周，不手填 `YYYY-Wxx`。
- Element Plus 单选 `ElSelect`，选项来自 `GET /api/v1/settings/cities` 中 `enabled=true` 的城市。
- “生成周报”按钮；未选择周或城市时禁止提交并显示提示。

前端将选中日期按 ISO 8601 周规则转换为 `YYYY-Wxx`。虽然界面只允许一个城市，仍兼容现有后端请求结构：

```json
{
  "week": "2026-W29",
  "cities": ["nb"]
}
```

后端增加请求校验，`cities` 必须且只能包含一个城市代码。列表中城市代码显示为配置中心的城市名称。

### 2.2 文件下载

每条已生成周报继续提供“Markdown”和“Excel”两个下载按钮。前端不再使用不携带认证头的 `window.open`，改为通过现有 Axios 客户端请求：

- `GET /api/v1/reports/{id}/download?format=md`
- `GET /api/v1/reports/{id}/download?format=xlsx`

请求使用 `responseType: "blob"` 并自动携带登录 Token；响应成功后根据 `Content-Disposition` 或周次生成文件名，通过临时 `<a download>` 保存文件，随后释放 Object URL。失败时显示 Toast。

## 3. 活动批量删除

### 3.1 前端交互

- 活动表格增加 Element Plus selection 列。
- 筛选工具栏增加带 `Delete` 图标的“批量删除”按钮。
- 没有勾选记录时按钮禁用。
- 点击后确认文案包含勾选数量，例如“确认删除选中的 3 条活动？”。
- 确认后提交一次批量请求；成功 Toast 展示实际删除数量，清空选择并刷新当前列表。
- 单条删除入口继续保留。

勾选范围仅限用户当前明确选择的行，不支持按筛选条件删除全部数据，也不跨页保留选择。

### 3.2 后端接口

新增 `DELETE /api/v1/activities/batch`：

```json
{
  "ids": [1, 2, 3]
}
```

约束与行为：

- `ids` 至少 1 个、最多 100 个，重复 ID 去重。
- 仅处理当前不是 `DELETED` 或 `MERGED` 的活动。
- 在单个数据库事务中把匹配记录状态更新为 `DELETED` 并刷新 `updated_at`。
- 返回 `deleted_ids` 和 `deleted_count`；若没有任何可删除记录则返回 404。
- 继续使用现有登录鉴权，不新增权限模型。

## 4. 错误处理

- OpenCLI 测试、周报生成、周报下载和批量删除均用 Toast 给出明确成功或失败反馈。
- 周报城市配置为空时，下拉框显示无数据状态，生成按钮不可用。
- 批量删除请求失败时保留当前选择，便于用户重试；只有成功后清空。
- 下载请求失败时不创建空文件，也不打开新窗口。

## 5. 测试与验收

### 组件测试

- 配置中心：测试期间出现 Loading 图标并禁用按钮，成功和失败后图标消失并触发 Toast。
- 周报管理：渲染周选择器、启用城市单选下拉；提交单城市数组；下载调用 Blob 接口。
- 活动管理：选择行后启用批量删除，确认后只提交选中 ID，成功后刷新。

### 后端测试

- 周报生成拒绝零城市和多个城市，接受单城市。
- 批量删除正常软删除、ID 去重、无有效 ID 返回 404、未登录拒绝访问。
- 现有 Markdown/XLSX 下载接口继续通过。

### 浏览器功能测试

- OpenCLI 测试请求挂起时显示 Loading，完成后隐藏并出现 Toast。
- 通过周选择器和城市下拉生成周报，并验证 Markdown、Excel 两个鉴权下载请求。
- 勾选多条活动、确认批量删除，验证请求体、成功 Toast 和列表刷新。

## 6. 非目标

- 不支持多城市周报。
- 不支持跨页批量选择或按当前筛选条件一键删除全部活动。
- 不改变周报 Markdown/XLSX 内容格式。
- 不新增新的 UI 库或自定义图标体系。
