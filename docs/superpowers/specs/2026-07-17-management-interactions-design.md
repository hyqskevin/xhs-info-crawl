# 配置测试、周报导出与活动批量删除设计

## 目标

在不改变一期 Vue 3、Element Plus、FastAPI、Celery、SQLite 技术栈的前提下，改善管理端交互与抓取可靠性：明确展示 OpenCLI 连接测试过程；以日期和城市配置选择周报范围并可靠下载文件；支持对勾选的活动执行批量软删除；细化抓取进度、隔离单笔记错误并支持失败任务续跑。

## 全局约束

- 前端继续使用 Element Plus 组件和 `@element-plus/icons-vue` 图标，不使用 Emoji，不自行实现已有基础 UI。
- 周报一次只允许选择一个城市，不支持多城市合并导出。
- 城市选项只来自配置中心中已启用的城市，不允许填写内部城市代码。
- 活动删除保持现有软删除语义，即状态更新为 `DELETED`，不物理删除数据库记录。
- 所有重试次数和间隔均通过 `.env` 配置，不在业务代码中写死。
- 所有新增交互同时补充组件测试、后端 API 测试和 Playwright 浏览器功能测试，并同步更新 `tests` 目录规格。

## 1. OpenCLI 测试状态

配置中心保留“测试 OpenCLI”按钮。点击后设置本地 `testingOpenCLI` 状态：

- 按钮在请求期间禁用，防止重复测试。
- 按钮右侧显示 Element Plus `Loading` 图标，并使用 `is-loading` 动画；测试结束后立即隐藏。
- 请求成功后显示成功 Toast：“OpenCLI 登录与连接正常”。
- 未登录时显示错误 Toast：“请在 Chrome 登录小红书”；其他错误优先展示接口返回原因，没有原因时展示连接测试失败提示。
- 无论成功或失败，都在 `finally` 中恢复按钮和加载图标状态。

该功能不改变现有 `POST /api/v1/settings/opencli/test` 接口。

## 2. 单城市周报生成与下载

### 2.1 生成条件

周报工具栏改为：

- Element Plus 周选择器 `ElDatePicker type="week"`，用户选择自然周，不手填 `YYYY-Wxx`。
- Element Plus 单选 `ElSelect`，选项来自 `GET /api/v1/settings/cities` 中 `enabled=true` 的城市。
- “生成周报”按钮；未选择周或城市时禁止提交并显示提示。

前端将选中日期按 ISO 8601 周规则转换为 `YYYY-Wxx`。虽然界面只允许一个城市，仍兼容现有后端请求结构：

```json
{
  "week": "2026-W29",
  "cities": ["nb"]
}
```

后端增加请求校验，`cities` 必须且只能包含一个城市代码。列表中城市代码显示为配置中心的城市名称。

### 2.2 文件下载

每条已生成周报继续提供“Markdown”和“Excel”两个下载按钮。前端不再使用不携带认证头的 `window.open`，改为通过现有 Axios 客户端请求：

- `GET /api/v1/reports/{id}/download?format=md`
- `GET /api/v1/reports/{id}/download?format=xlsx`

请求使用 `responseType: "blob"` 并自动携带登录 Token；响应成功后根据 `Content-Disposition` 或周次生成文件名，通过临时 `<a download>` 保存文件，随后释放 Object URL。失败时显示 Toast。

## 3. 活动批量删除

### 3.1 前端交互

- 活动表格增加 Element Plus selection 列。
- 筛选工具栏增加带 `Delete` 图标的“批量删除”按钮。
- 没有勾选记录时按钮禁用。
- 点击后确认文案包含勾选数量，例如“确认删除选中的 3 条活动？”。
- 确认后提交一次批量请求；成功 Toast 展示实际删除数量，清空选择并刷新当前列表。
- 单条删除入口继续保留。

勾选范围仅限用户当前明确选择的行，不支持按筛选条件删除全部数据，也不跨页保留选择。

### 3.2 后端接口

新增 `DELETE /api/v1/activities/batch`：

```json
{
  "ids": [1, 2, 3]
}
```

约束与行为：

- `ids` 至少 1 个、最多 100 个，重复 ID 去重。
- 仅处理当前不是 `DELETED` 或 `MERGED` 的活动。
- 在单个数据库事务中把匹配记录状态更新为 `DELETED` 并刷新 `updated_at`。
- 返回 `deleted_ids` 和 `deleted_count`；若没有任何可删除记录则返回 404。
- 继续使用现有登录鉴权，不新增权限模型。

## 4. 错误处理

- OpenCLI 测试、周报生成、周报下载和批量删除均用 Toast 给出明确成功或失败反馈。
- 周报城市配置为空时，下拉框显示无数据状态，生成按钮不可用。
- 批量删除请求失败时保留当前选择，便于用户重试；只有成功后清空。
- 下载请求失败时不创建空文件，也不打开新窗口。

## 6. 非目标

- 不支持多城市周报。
- 不支持跨页批量选择或按当前筛选条件一键删除全部活动。
- 不改变周报 Markdown/XLSX 内容格式。
- 不新增新的 UI 库或自定义图标体系。

## 7. 抓取鲁棒性与单笔记隔离

### 7.1 已确认根因

任务 4 搜索发现 113 条候选笔记，前 5 条完成下载、OCR、MiniMax 提取和归档。第 6 条的 MiniMax 结构化结果返回 `start_time: "4/5"`，任务层直接调用 `datetime.fromisoformat("4/5")`，触发 `Invalid isoformat string`。当前实现把单笔记异常抛到整个 Celery 任务，导致任务失败且 `failed_notes` 没有增加。

### 7.2 模型输出归一化

MiniMax 输出在进入数据库模型前必须经过统一归一化，不允许任务层直接解析未验证字符串：

- 接受完整 ISO 日期时间、`YYYY-M-D`、`M/D`、`M月D日` 及可选时分。
- 缺少年份时使用本次任务执行时间所在年份；生成 ISO 日期时间字符串。
- 无效日期、空字符串或无法识别的值归一化为 `None`，活动状态设为 `NEEDS_REVIEW`，不抛出导致任务中断的异常。
- `end_time` 使用同一规则；结束时间早于开始时间时置空并进入待完善。
- `confidence`、`source_image_indexes` 和文本字段继续做类型、范围和去重校验。
- MiniMax 提示词明确要求 ISO 8601，但系统不依赖提示词保证数据正确。

### 7.3 阶段重试与降级

新增环境变量：

- `PIPELINE_STAGE_MAX_RETRIES=2`：详情读取、图片下载、OCR 和 MiniMax 单阶段最大尝试次数。
- `PIPELINE_STAGE_RETRY_DELAY_SECONDS=2`：阶段重试间隔秒数。

处理规则：

- 详情读取和图片下载遇到临时 OpenCLI 错误时按配置重试；登录失效仍立即暂停整个任务。
- 单张图片 OCR 失败时记录该图片错误并继续处理其他图片；全部 OCR 失败时仍可使用标题和正文提取。
- MiniMax 遇到超时、5xx 或无效结构化输出时重试；最终失败后降级为规则提取，并把活动标记为待完善。
- 每条笔记使用独立事务边界。最终失败时只回滚该笔记，`failed_notes += 1`，记录包含来源 URL、阶段和原因的任务日志，然后继续下一条。
- 搜索结果先按来源 URL 去重，避免同一次任务重复处理相同笔记。

只有登录、数据库连接或 Celery/文件系统等任务级基础设施异常才停止整个任务。单笔记失败不会把任务状态改为 `FAILED`；处理结束且存在失败笔记时状态为 `COMPLETED_WITH_ERRORS`。

## 8. 细化任务进度

### 8.1 数据模型

在 `crawl_tasks` 增加以下字段并提供 Alembic 迁移：

- `current_stage`：`SEARCHING`、`DOWNLOADING`、`OCR`、`EXTRACTING`、`ARCHIVING` 或空值。
- `downloaded_notes`：完成详情和图片下载的笔记数。
- `ocr_notes`：完成 OCR 阶段的笔记数；没有图片也视为完成该阶段。
- `extracted_notes`：完成活动提取和归档的笔记数。
- `current_note`：当前处理的笔记标题，允许为空。

现有字段语义固定为：

- `total_notes`：搜索发现并按 URL 去重后的候选笔记总数。
- `success_notes`：完成全部处理并归档的笔记数，与 `extracted_notes` 保持一致。
- `failed_notes`：经过阶段重试和降级后仍无法完成的笔记数。

整体进度使用 `(success_notes + failed_notes) / total_notes`；搜索阶段总数未知时显示不确定进度，不伪造百分比。

### 8.2 界面展示

任务日志表格把“发现、成功、失败”扩展为：发现、已下载、OCR 完成、提取完成、失败，并显示当前阶段和进度条。日志继续提供逐阶段错误详情。

仪表盘增加“最近抓取任务”卡片，每 3 秒轮询仪表盘摘要：

- 显示任务 ID、中文状态、当前阶段、当前笔记、五项计数和整体进度条。
- `RUNNING`、`DOWNLOADING` 等执行状态展示动态进度。
- `COMPLETED` 和 `COMPLETED_WITH_ERRORS` 展示最终结果。
- `FAILED` 时展示错误原因与“继续抓取”按钮。
- 页面卸载时停止轮询，避免后台请求泄漏。

## 9. 失败任务续跑

新增 `POST /api/v1/tasks/{task_id}/restart`，只允许状态为 `FAILED` 的任务调用：

- 复用原任务 ID、城市、关键词、时间和博主参数，不创建重复任务。
- 保留已经成功入库和归档的数据及成功计数，清除任务级错误、完成时间和当前阶段后重新入队。
- 重新搜索后按来源 URL 跳过已经成功保存的笔记，只处理未完成来源。
- 续跑期间禁止再发起其他抓取任务，也禁止重复点击续跑。
- 若任务不是 `FAILED`、已有任务正在执行或原城市配置已停用，返回明确的 409/422 错误。

任务日志追加“任务继续抓取”和续跑后的阶段记录，不删除历史失败日志，便于追踪。

## 10. 鲁棒性与进度测试

### 后端测试

- `4/5`、`4月5日`、完整 ISO 日期均被归一化；非法日期变为 `None/NEEDS_REVIEW`。
- MiniMax 临时失败会按配置重试，最终失败降级为规则提取。
- 单笔记失败增加失败计数并继续下一条，最终状态为 `COMPLETED_WITH_ERRORS`。
- 各阶段计数、当前阶段和整体完成计数按真实处理顺序更新。
- 失败任务续跑复用任务 ID，跳过已成功来源；非失败任务和并发任务不能续跑。

### 组件与浏览器测试

- 任务日志展示细化计数、中文阶段和进度条。
- 仪表盘轮询显示阶段进度，卸载时停止轮询。
- 失败任务显示继续抓取按钮；点击后调用续跑接口、按钮进入加载状态并刷新任务。
- 单笔记失败但任务继续时，界面最终展示“完成但有错误”及准确失败数量。
