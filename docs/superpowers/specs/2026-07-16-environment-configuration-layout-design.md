# 环境变量分类与注释设计

## 目标

让开发者无需阅读源码即可理解每个环境变量的用途，同时保持现有本地配置值和运行行为不变。

## 设计

- 同步整理根目录 `.env` 与 `.env.example`。
- 按应用安全、管理员、后端、前端、数据文件、Celery、定时抓取、小红书浏览器、MiniMax、PaddleOCR 分类。
- 每个变量前提供中文说明；有单位、枚举或安全要求时明确写出。
- `.env` 保留全部当前值；`.env.example` 只保留安全示例或空的敏感项。
- 不新增、删除或重命名环境变量，不改变应用运行逻辑。

## 验证

对整理前后的变量名集合和值逐项比较，并运行后端配置加载检查和前端构建检查。
