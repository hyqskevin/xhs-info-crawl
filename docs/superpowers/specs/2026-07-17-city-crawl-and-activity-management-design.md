# 城市抓取配置与活动管理设计

## 目标

配置中心按城市统一维护关键词和小红书原生时间范围；活动管理只处理爬取结果，提供友好的筛选、分页和人工审核能力。

## 配置中心

- 城市是主配置，新增和编辑表单包含城市名称、多个关键词、抓取时间范围和启用状态。
- 抓取时间范围仅允许小红书原生选项：不限、一天内、一周内、半年内；每个城市独立设置，默认一周内。
- 城市 `code` 继续作为数据库关联键，但由后端根据城市名称自动生成，前端不展示、不要求输入。
- 移除独立“关键词”标签页。博主配置保留，并增加编辑能力；博主关联城市使用城市下拉选择。
- 表格使用中文列名和中文启用状态，不显示数据库 `id`、内部 `code` 或原始布尔值。

## 爬虫

- 抓取任务以城市配置为准，逐城市读取其启用关键词和时间范围。
- OpenCLI 搜索筛选点击该城市配置的小红书原生时间范围；“不限”不点击时间选项。
- 完整活动也统一写入 `RAW`，界面显示“待审核”；字段不完整写入 `NEEDS_REVIEW`，显示“待完善”。
- 只有人工改为 `APPROVED`（已通过）的活动进入周报。

## 活动管理

- 移除“新增活动”入口和新增弹窗，只允许查看、编辑、删除爬取结果。
- 城市筛选使用城市名称下拉，API 仍接收内部城市 code。
- 增加活动开始时间范围筛选，后端使用 `start_date`、`end_date`。
- 保留状态筛选但显示中文：待完善、待审核、已通过、已发布。
- 分页允许每页 10、20、50、100 条；筛选或每页数量改变时回到第一页。
- 表格将城市代码映射为城市名，时间使用本地可读格式。

## 数据迁移与兼容

- `cities` 增加 `recent_filter` 字段，已有城市迁移为“一周内”。
- 现有关键词表继续保存城市与关键词的一对多关系，避免破坏任务和历史数据。
- 旧接口的关键词读写能力可保留供内部兼容，但前端不再提供独立管理入口。

## 验证

- 后端覆盖城市组合新增/编辑、自动 code、时间范围校验、抓取筛选和活动分页/日期筛选。
- 前端组件与 Playwright 覆盖城市组合配置、编辑、城市下拉、时间筛选、分页和无新增活动入口。
- 同步更新 spec、UI 和 tests Markdown 文档。

## 仪表盘抓取入口与任务日志

- 抓取属于执行功能，入口放在仪表盘；任务日志页面只负责监控历史任务和查看日志。
- 仪表盘从配置中心读取选项：城市单选、该城市关键词多选、该城市博主多选、小红书原生时间范围单选。
- 切换城市后清空之前城市的关键词和博主选择，并将时间范围重置为该城市默认值。
- 至少选择一个关键词或博主才允许提交；任务参数保存内部城市 code、关键词、博主 ID 和本次时间范围。

## 去重完成后的数据状态

- 去重候选列表默认仅显示 `pending`，保留 A/B 后立即从列表消失。
- 保留活动维持原审核状态，不因去重自动变成已通过。
- 未保留活动标记为 `DELETED`，不会再出现在活动管理中；候选记录保留为审计历史但不在待审核列表显示。
