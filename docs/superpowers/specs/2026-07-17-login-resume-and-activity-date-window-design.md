# 登录续跑与活动日期窗口设计

## 1. 背景与已确认规则

当前抓取任务在 OpenCLI 检测到小红书登录态失效后会进入 `PAUSED`，但仪表盘没有恢复入口。用户只能看到“请在 Chrome 登录小红书后重试”，无法打开登录页，也无法从页面继续原任务。

任务 4 同时产生了 2022、2024、2025 年的活动。数据库核对结果表明，这些记录来自本次抓取，而不是前端格式化错误。根因有两层：

1. 小红书的“一周内”筛选约束的是笔记发布时间，并不保证笔记中描述的活动仍然有效。
2. 提取流水线只校验日期能否解析，没有校验活动是否处于业务有效期；MiniMax 返回的历史年份因此被直接保存为 `RAW`。

本次已确认以下产品规则：

- 已结束的历史活动不入库，并清理数据库中现有的过期活动。
- 活动有效窗口为抓取任务开始日（含）至未来第 60 天（含）。
- 一周内、一天内等城市配置继续表示小红书笔记发布时间范围，不改为活动时间范围。
- 登录恢复复用原任务 ID、原参数和已完成进度，不新建任务。
- 阶段一仍保持本地运行，不引入 Redis、MinIO 或 Docker。

## 2. 方案选择

### 2.1 活动日期

采用“后端入库硬校验”方案。MiniMax 提示词负责减少错误，后端校验器是最终可信边界：

- 可解析且与 60 天窗口相交的活动允许入库。
- 可解析但完全位于窗口之前或窗口之后的活动直接跳过，并写任务日志。
- 无法确定日期的活动不再用当前时间兜底，保留为 `NEEDS_REVIEW`，开始时间为空，且不进入周报。

不采用仅前端隐藏方案，因为该方案仍会污染 SQLite、Markdown 和 Excel；也不采用把所有越界活动保留为待审核的方案，因为用户已确认历史活动和超过 60 天的远期活动不需要保留。

### 2.2 登录恢复

采用显式两步流程：

1. “打开小红书登录”负责唤醒本机 Chrome 并打开小红书登录页面。
2. “检测登录并继续”先通过 OpenCLI 检查登录态；成功后才把同一任务重新入队。

不采用固定延时后自动恢复，因为用户完成扫码或验证码的时间不可预测；也不采用自动轮询登录态，因为后台持续调用浏览器会造成不透明行为和额外失败噪声。

## 3. 活动日期有效性设计

### 3.1 统一参考时间

- 参考日取任务第一次执行的 `started_at`，转换为 `Asia/Shanghai` 后的自然日。
- 任务暂停后续跑时不得覆盖原 `started_at`，确保同一任务多次运行使用同一窗口。
- 窗口定义为 `[reference_date 00:00:00, reference_date + 60 days 23:59:59]`，两端均包含。
- 60 天配置写入 `.env`：`ACTIVITY_FUTURE_WINDOW_DAYS=60`，并在 `.env.example` 中按“活动日期校验”分类说明。

### 3.2 日期归一化

`normalize_activity_datetime` 继续支持 ISO、`YYYY-M-D`、`M/D`、`M月D日` 等输入，并增加参考日参数：

- 输入明确包含年份时保留原年份，不擅自改年。
- 输入不含年份时，先使用参考日所在年份；若该日期已早于参考日，则尝试下一年。
- 推断后的日期只有落入未来 60 天窗口才视为有效，否则按越界处理。
- `晚间`、`待定`、非法日期等不能可靠转换的内容返回空值，不抛出任务级异常。
- MiniMax 提示词同时传入参考日期、时区和 60 天上限，并要求不要猜测正文未出现的年份；后端仍必须复核模型结果。

### 3.3 窗口判定

新增独立的活动窗口判定函数，便于单元测试：

- 只有开始时间：开始时间落在窗口内则保留，否则跳过。
- 同时有开始和结束时间：活动时间段与窗口有交集则保留；这允许已经开始但尚未结束的长期展览继续存在。
- 结束时间早于开始时间：结束时间清空，按只有开始时间处理并记录警告。
- 开始时间缺失：保留为 `NEEDS_REVIEW`，不参与窗口外删除，也不进入周报。
- 明确日期完全早于窗口或晚于窗口：不创建 `Activity`，任务的 `skipped_notes` 不增加，新增活动级 `skipped_activities` 进度计数并记录活动名称、原日期和跳过原因。

为正确表达未知日期，`activities.start_time` 改为可空；API、活动列表和详情统一显示“待确认”。已有按时间排序改为“有效日期升序、空日期最后”。

### 3.4 当前脏数据清理

实现一次性清理脚本，并在本次升级中实际执行：

1. 使用执行当天的 `Asia/Shanghai` 日期和未来 60 天窗口。
2. 仅处理爬虫生成的活动（`note_id IS NOT NULL`）；无日期的待审核记录不删除。
3. 删除完全位于窗口外的活动前，先删除关联的去重候选，避免孤儿记录。
4. 保留原始笔记、正文、OCR、来源图片和 `source.md`，便于后续审计，不因活动过期而删除原始证据。
5. 按受影响任务重新生成 `activities.md` 与 `activities.xlsx`，使文件、数据库和前端结果一致。
6. 输出清理摘要：扫描数、删除数、保留数、受影响任务 ID；脚本重复执行应得到相同结果。

清理脚本不在每次应用启动时自动运行，避免系统日期或配置误改时静默删除数据。持续正确性由新入库校验保证。

## 4. 登录与续跑设计

### 4.1 打开登录页

新增管理员接口：

```text
POST /api/v1/settings/opencli/open-login
```

接口调用独立的本机浏览器启动服务：

- macOS 使用已配置的 Google Chrome 打开小红书主站。
- Windows 和 Linux 使用对应的平台启动方式，仍优先 Chrome。
- 登录地址来自 `.env` 的 `XHS_LOGIN_URL=https://www.xiaohongshu.com/explore`。
- 浏览器名称来自 `.env` 的 `XHS_LOGIN_BROWSER=Google Chrome`。
- 启动成功返回 200；Chrome 不存在或启动失败返回 503，并把可手动复制的登录地址返回给前端。
- 接口只在本地管理员登录后可调用，不读取、返回或记录 Cookie。

OpenCLI 没有独立登录窗口；它继续复用 Chrome 中的小红书登录态。

### 4.2 检测登录并继续

沿用现有接口：

```text
POST /api/v1/tasks/{task_id}/restart
```

接口扩展为接受 `PAUSED`：

- 对 `FAILED`、`STOPPED` 保持现有逻辑。
- 对 `PAUSED` 先调用 `OpenCLIAdapter.check_login()`。
- 检测失败时返回 409 和 `AUTH_REQUIRED`，任务保持 `PAUSED`，进度与错误信息不变。
- 检测成功后把任务改为 `PENDING`，清除登录错误和当前阶段，再把原任务 ID 投递给 Celery。
- `total_notes`、下载/OCR/提取/成功/失败/跳过计数均保留；已完成笔记继续通过来源 URL 去重跳过。
- 同一时间仍只允许一个活动抓取任务执行。

### 4.3 仪表盘交互

任务状态为 `PAUSED` 时，在任务卡片中显示：

- `打开小红书登录`：Element Plus `ElButton`，使用 Element Plus 图标；请求期间显示按钮 loading，成功或失败后显示 Toast。
- `检测登录并继续`：主按钮，请求期间显示 loading；登录未完成时显示明确 Toast；成功后立即刷新任务状态和进度。

不使用 emoji，不自行绘制图标。`FAILED`、`STOPPED` 的“继续抓取”和运行任务的“安全停止”保持原行为。

## 5. 数据流与错误隔离

```text
搜索近一周笔记
  -> 标题关键词校验
  -> 下载正文和图片
  -> OCR
  -> MiniMax 多活动提取
  -> 日期归一化
  -> 60 天窗口校验
     -> 有效：入库、去重、归档
     -> 明确越界：记录日志并跳过该活动
     -> 日期未知：NEEDS_REVIEW，日期为空
```

一条笔记中的某个活动越界，只跳过该活动；同笔记的其他有效活动继续入库。一条笔记处理失败仍沿用现有隔离机制，不影响后续笔记。登录失效属于任务级暂停，不计为笔记失败。

## 6. 测试设计

### 6.1 后端自动化

- 日期归一化：明确年份、无年份跨年、非法日期、`晚间`、边界第 0 天和第 60 天。
- 窗口判定：历史活动、未来 61 天、窗口内活动、跨窗口长期活动、未知日期。
- 提取流水线：同一笔记包含有效、过期、远期、未知日期活动时分别处理，单项失败不影响其他项。
- 入库：未知日期不再回填当前时间，状态为 `NEEDS_REVIEW`。
- 清理脚本：删除越界爬虫活动及关联去重候选，保留笔记/图片，重建 Markdown 与 Excel，重复执行幂等。
- 登录页接口：模拟各平台启动成功和失败，不在测试中真实打开 Chrome。
- 任务续跑：`PAUSED` 未登录保持暂停，已登录复用原任务 ID、原 `started_at` 和原进度。

### 6.2 前端组件与浏览器功能测试

- `PAUSED` 卡片显示两个登录恢复按钮，其他状态不显示。
- 两个按钮分别验证 loading、Toast、失败恢复和成功刷新。
- 活动列表、详情正确显示空日期为“待确认”，分页和时间筛选不受影响。
- 浏览器 E2E 使用 API stub 验证完整暂停恢复流程，不依赖真实小红书账号。

### 6.3 真实本地验证

- 点击仪表盘按钮应打开 Chrome 小红书主站。
- 人工登录后点击“检测登录并继续”，OpenCLI `whoami` 通过，暂停任务变为等待/抓取中。
- 用一个包含历史活动和未来 60 天内活动的真实笔记验证：历史活动不入库，有效活动进入 SQLite、Markdown 和 Excel。
- 执行清理脚本后，活动管理中不再显示当前窗口之外的已知日期活动。
